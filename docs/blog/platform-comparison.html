The honest verdict

n8n wins on AI and cost at scale. If your workflow needs LLM processing — scoring, summarizing, drafting, classifying — n8n's native LangChain nodes and Code node make it the only serious option. Self-hosted n8n at $6/month handles thousands of AI-powered executions that would cost $100+ in Zapier.

Make wins on visual clarity and batch operations. The Router module (parallel branches) is genuinely better than anything Zapier or n8n offers for splitting flows. The Iterator + Aggregator pattern for batch processing is unmatched. The scenario editor is the most intuitive visual builder available.

Zapier wins on integrations and accessibility. 6,000+ apps means you'll always find the connector you need. Non-technical team members can build and modify Zaps without developer help. For simple, linear automations that don't need AI or batch processing, Zapier's speed of setup often beats the alternatives.

The real answer: Most production automation stacks use more than one platform. The right question isn't "which platform?" — it's "which platform handles which segment of this pipeline best?"

My personal recommendation framework

Does the workflow need AI/LLM processing?
├── YES → n8n (cheapest, most capable)
└── NO → continue ↓

Is the team non-technical?
├── YES → Zapier (they can maintain it themselves)
└── NO → continue ↓

Does it process batches or high volumes (>1,000 items/mo)?
├── YES → Make (cheapest per-operation pricing)
└── NO → continue ↓

Do you need a specific app integration not in n8n or Make?
├── YES → Zapier (6,000+ apps)
└── NO → Make (best value for mid-complexity workflows)
